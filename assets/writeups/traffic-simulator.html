Multi-Lane Traffic Simulation with Entrances and Exits

Particle-based traffic model · Python · Object-Oriented Design · Unit Testing

Python OOP Simulation Unit Testing Traffic Modeling Emissions (VSP)

This project focuses on building a discrete, particle-based traffic simulator capable of modeling real-world highway dynamics, including multi-lane roads, entrance ramps, exit ramps, and heterogeneous driver behaviors.

The simulator serves as a virtual laboratory for exploring congestion patterns, safety implications, and future extensions such as emissions modeling and policy analysis.

GitHub Repo

Problem Statement

Traffic congestion is a persistent urban challenge that contributes to increased travel times, higher emissions, safety risks, and socioeconomic inequities. Traditional traffic models often simplify driver behavior or fail to explicitly capture the disruptive effects of entrance and exit ramps.

The goal of this project is to design a flexible simulation framework that models individual vehicle decisions and their emergent effects on traffic flow.

Approach

I implemented a discrete-time, particle-based model where each vehicle is treated as an autonomous agent with a current state and a set of possible actions.

  • Object-oriented design for cars, lanes, entrances, and exits
  • Rule-based acceleration, braking, merging, and lane-change logic
  • Explicit modeling of entrance-induced disturbances and exit relief
  • Robust unit tests to validate correctness under edge cases

Simulation Details

The simulation evolves traffic discretely over time, updating vehicle positions, velocities, and lane assignments at each timestep. Inputs such as road geometry, entry rates, and driver behavior parameters can be modified to explore different scenarios.

Results & Observations

  • Successfully modeled congestion waves caused by merging behavior
  • Demonstrated relief effects downstream of exit ramps
  • Verified stability and correctness through extensive automated tests

Extensions & Future Work

  • Calibration with real-world traffic data (e.g., Caltrans loop detectors)
  • Integration with emissions models using Vehicle Specific Power (VSP)
  • Equity and policy scenario analysis

Key Takeaways

This project strengthened my ability to translate real-world systems into modular, testable simulation code. It also highlighted the importance of behavioral modeling and domain knowledge when interpreting computational results.
