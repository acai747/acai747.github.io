Playlist “Vibe” Anomaly Detection + Recommendations

Anomaly Detection · Similarity Search · Feature Standardization · PCA (Visualization) · Python

Python Pandas NumPy Scikit-learn Cosine Distance Z-scores PCA

Built a content-based system that identifies “off-vibe” songs within user-curated playlists by comparing song audio features (e.g., energy, tempo, danceability, valence) to a playlist’s dominant musical profile.

The system flags anomalies relative to the playlist context (not globally) and recommends replacements that better match the playlist’s core vibe.

GitHub Repo

Problem Statement

Curated playlists are often built around a specific mood or “vibe.” A single mismatched track can disrupt the flow and reduce engagement, but most recommendation systems focus on predicting what users might like next rather than validating internal playlist coherence.

The goal of this project is to detect songs that deviate from the dominant vibe within a playlist and suggest replacements that better match the playlist identity.

Data

The project used two inputs: (1) user-created playlists (artist/title metadata) and (2) a larger song-level dataset containing Spotify-derived audio features (tempo, loudness, danceability, energy, acousticness, instrumentalness, etc.).

  • Playlist-level inputs: song titles + artists (inconsistent formatting and missing matches)
  • Song feature table: numeric audio features used to represent each track’s musical profile

Approach

Each playlist is represented in standardized feature space. I compute a playlist centroid as the dominant vibe and measure each song’s deviation using cosine distance. Songs with unusually high deviation are flagged as anomalies.

  • Cleaned and normalized track/artist text for improved matching
  • Standardized audio feature vectors using StandardScaler
  • Computed a playlist centroid (dominant vibe representation)
  • Calculated cosine distance of each track to the centroid
  • Converted distances to z-scores to flag outliers above a threshold

Recommendation Strategy

For each detected anomaly, the system proposes replacement tracks that are closer to the playlist centroid in the same standardized feature space. This produces interpretable recommendations aligned with the playlist’s dominant vibe.

  • Compute similarity between playlist centroid and candidate tracks
  • Rank replacements by smallest cosine distance to centroid
  • Return top-N replacements and explain why (feature alignment)

Results & Observations

  • Worked well for playlists with a single dominant vibe (clear centroid structure)
  • PCA plots (when used) visually supported clusters and outlier tracks
  • Anomaly scores were interpretable and aligned with audible “mismatch” in many playlists

Challenges & Limitations

  • Track matching issues (aliases, formatting differences, incomplete playlist metadata)
  • Small playlists provide less structure, making anomaly detection less stable
  • Multi-vibe playlists can be misrepresented by a single centroid approach
  • Audio features may not fully capture subjective vibe (context/lyrics/culture)

Extensions & Future Work

  • Multi-centroid representations (e.g., cluster playlists into sub-vibes)
  • Robust outlier detection methods (e.g., Isolation Forest, LOF) for comparison
  • Hybrid features using lyrics/text embeddings or genre metadata
  • User feedback loop to evaluate recommendation acceptance

Key Takeaways

This project demonstrates how simple, interpretable similarity-based methods can fill a gap in music tooling by focusing on playlist coherence rather than global recommendation. It strengthened my experience in feature engineering, anomaly detection, similarity metrics, and building practical systems that align model outputs with user experience.
