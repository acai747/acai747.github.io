Sentiment Analysis of Airline Reviews (VADER vs RoBERTa)

NLP · Sentiment Modeling · Baselines vs Transformers · Evaluation · Python

Python Pandas NLTK (VADER) Hugging Face RoBERTa Model Evaluation

Built a sentiment analysis workflow on 20K+ airline reviews and compared a lightweight lexicon baseline (VADER) against a transformer model (RoBERTa) to understand performance, robustness, and deployment tradeoffs.

GitHub Repo Report (Optional)

Problem Statement

Airlines receive large volumes of text feedback across service, delays, comfort, and staff experiences. A sentiment classifier can help convert unstructured reviews into measurable signals for operations and customer experience teams.

The goal of this project was to build a sentiment pipeline and evaluate whether a modern transformer model meaningfully improves over a simple baseline, especially under real-world constraints like speed, interpretability, and cost.

Data

Used a dataset of 20,000+ airline reviews (from airlinequality.com) containing free-text feedback and associated labels/ratings.

  • Primary input: review text
  • Optional signals: rating fields / categories (if available)
  • Challenges: sarcasm, mixed sentiment, domain-specific wording, varying review lengths

Approach

I implemented two sentiment approaches and compared them under the same dataset and evaluation framework:

  • VADER (NLTK): lexicon-based sentiment scoring, lightweight and fast
  • RoBERTa (Hugging Face): transformer-based sentiment classification with richer context understanding

Evaluation

Models were evaluated using standard classification metrics and qualitative error review to understand failure modes.

  • Metrics: accuracy, precision, recall, F1, AUC (if applicable)
  • Additional checks: confusion matrix review, edge cases (mixed sentiment, sarcasm)
  • Operational lens: inference speed, cost, and ease of deployment
VADER Fast baseline · interpretable scoring · weaker on nuance
RoBERTa Better context handling · stronger nuance · higher compute cost

Key Findings

  • Transformer models generally handled context, negation, and mixed sentiment better than lexicon scoring
  • VADER performed well as a quick baseline and for high-volume, low-latency monitoring
  • Error analysis highlighted cases where both models struggle (sarcasm, ambiguous tone)

Challenges & Limitations

  • Subjective language and mixed sentiment within a single review
  • Domain-specific vocabulary that may not be captured by generic sentiment resources
  • Transformer inference cost/latency vs baseline simplicity

Extensions & Future Work

  • Fine-tune a domain-adapted transformer on airline-specific review labels
  • Aspect-based sentiment (e.g., delays vs staff vs comfort) for more actionable insights
  • Human-in-the-loop labeling for difficult or ambiguous cases
  • Deploy as a dashboard feed with trend monitoring by airline/route/time

Key Takeaways

This project strengthened my applied NLP skills and ability to compare baselines vs deep learning approaches. It also reinforced a product mindset: model choice should reflect operational constraints (speed/cost) and stakeholder needs (interpretability/actionability), not just raw performance.
