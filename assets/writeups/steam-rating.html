Steam Game Rating Prediction (Binary Classification)

Classification · Feature Engineering · Model Comparison · scikit-learn · TensorFlow

Python Pandas scikit-learn TensorFlow Feature Engineering Binary Classification

Built a binary classifier to predict whether a Steam game would receive a positive rating based on structured attributes such as platform support, pricing/discount signals, and game tags.

GitHub Repo

Problem Statement

Steam hosts thousands of games with varying quality, pricing strategies, and platform availability. Predicting whether a game is likely to be positively rated can help with product analysis, merchandising strategy, and understanding what features correlate with customer satisfaction.

The goal of this project was to build an interpretable binary classification workflow and compare traditional ML methods with a neural network approach.

Data

Used a dataset of 50,000+ cleaned and preprocessed Steam game records containing game attributes and review outcomes.

  • Size: 50K+ records
  • Inputs: platform support (Windows/Mac), price/discount flags, tags/genres, metadata fields
  • Target: binary label indicating whether the game receives a positive rating

Approach

I built a supervised learning workflow including preprocessing, feature engineering, model training, and evaluation.

  • Preprocessed structured fields and encoded categorical attributes (e.g., tags)
  • Engineered useful features from pricing and discount metadata
  • Trained and evaluated multiple classification models for comparison
  • Compared a baseline ML approach vs a deep learning approach (TensorFlow)

Models

  • Deep learning: feedforward neural network in TensorFlow
Baseline Goal Strong interpretability + solid performance with simpler features
NN Goal Capture non-linear interactions across metadata and tags
Outcome Compared performance, generalization, and deployment tradeoffs

Results & Observations

  • Identified which features most strongly correlate with positive ratings (e.g., tags, pricing signals, platform support)
  • Documented performance tradeoffs between classical ML and neural networks
  • Validated generalization on held-out data and avoided overfitting through evaluation discipline

Challenges & Limitations

  • High-cardinality categorical features (game tags) required careful encoding
  • Potential label noise due to review manipulation or small sample sizes per game
  • Strong interaction effects between genre, price, and platform availability

Extensions & Future Work

  • Use text review content (NLP) to augment structured features
  • Calibrate probabilities to support business decisions (risk-aware thresholds)
  • Build separate models by genre or price tier to reduce heterogeneity
  • Deploy as a dashboard tool for product/market analysis

Key Takeaways

This project strengthened my end-to-end supervised learning workflow skills, especially feature engineering on real-world metadata and comparing classical ML with deep learning approaches. It also reinforced the importance of model interpretability and selecting evaluation metrics aligned to the business question.
